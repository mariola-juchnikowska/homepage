HTML

HTML to skrót od HyperText Markup Lanaguage. Jest to język znaczników, a nie programowania. Składa się ze znacznikó, tagów, elementów.

Budowa znacznika:

<znacznik atrybut="wartość">treść</znacznik>

semantyczny= mający znaczenie

Znaczniki
  • <main> - główna część dokumnetu
  • <section> - sekcje dokumentu, pod względem logicznego podziału
  • <article> - samodzielne treści, artykuły w gazecie
  • <header> - nagłówek dokumentu, sekcji, artykułu
  • <footer> - stopka dokumnetu, sekcji, artykułu
  • <h1-h6> - tesktowe nagłówki tytułowe- dokumentu, sekcji, artykułu
  • <p> - paragraf, akapit

CSS

CSS to Cascading Style Sheets- kaskadowe arkusze stylów. Odpowiadają za prezentacje, wygląd elementow HTML na ekranie, wydrukach, czytnikach. Ma swoje reguły.

Budowa reguły CSS:

selector {
właściowść: wartość;
}

Sposoby stylowania:
  • w pliku html- w ramach znacznika <head> wpisujemy znacznik <style>
  • w pliku css- dzięki zewnętrznemu skoroszycie stylów <link href="style.css" rel="stylesheet" type="text/css">, co umieszczamy w <head> pliku index.html

Tydzień 3

Kanban- Japońskie planowanie

Ta metodologia należy do zwinnego podejscia organizacji pracy, jest po to aby śledzić swoje działania w projekcie jak i może być formą motywacji, gdy codziennie będziemy mogli zadania przerzucać. Narzędziem do tego może być np. Trello.

  • TO DO- lista to zrobienia
  • IN PROGRESS- lista zadań aktualnie realizowanych, zalecane jest aby zajmować się jednym zadaniem w danym czasie
  • DONE- lista zadań które zostały zrealizowane

często w projektach IT jest również TESTING- czyli zadania oddane do testów. Dobrą praktyką jest tworzenie również DoD- czyli Definition of Done, na podstawie zadań.

HTML II- atrybuty i znaczniki

  • <br/>- przenosi treść do nowej lini, jest to znacznik samozamykający
  • <b>- pogrubienie tekstu- wyróżnik wizualny
  • <strong>- pogrubienie tekstu, podkreśla wysoką rangę danego tekstu- wyróżnik semantyczny
  • <i>- pochylenie tekstu- wyróżnik wizualny
  • <em>- podkreśla inne znaczenie tekstu, np. ironię- wyróżnik semantyczny
  • <a href="" target="">- linki zewnętrzne/łącza wewnątrz dokumentu
  • <element id="">- identyfikuje element, przydaje się do nawigacji w obrębie dokumentu,np.<a href="#nazwa-id">
  • <nav>- nawigacja po dokumencie
  • <img src="" alt="">- obrazek z adresem url, a także alt- czyli podpis obrazka, wyświetlany alternatywnie w przeglądarce

Znacznik, który powtarza się na każdej stronie powinien być wyciągnięty z <main>. Znacznik <nav>powinien być zawarty w <header>

CSS II

styl przy znaczniku z wykorzystaniem atrybutu style nazywamy stylem inline
<znacznik style="color:yellow;">treść</znacznik>. Nie zaleca się takiego stylowania, bo wprowadza chaos w pliku, czytelniej jest używać selektórów w odrębnym pliku style.css

Selektory
  • .class - to selektor klasy
  • element - to byt w htmlu, np h1, header
  • #id- powinno być unikalne

Istnieje również !important- nie powinniśmy tego nadużywać, bo może wprowadzić chaos w kodzie. Zapisujemy go za daną właściwością.

Klasyfikacja medalowa CSS
      !important
      style=""
      #id
      .class
      element

Gdy wystąpi systuacja, że medale wyniosą po równo, pierwszeństwo ma selector umieszczony niżej w pliku .css. Unikajmy zagnieżdżeń, np. zamiast nav a, użyjmy mu klase .header-link.

Jak nazywać klasy?
  • staraj się używać pojedynczych słów z j.ang.
  • jak nazwa jest dłuższa to użyj kebeb-case
  • staraj się jasno określać co zawiera klasa
  • nie zaczynaj od cyfry

Jak działa przelądarka

HTTP - Hypertext Transfer Protocol - protokół, którego używa przeglądaka
Jest również HTTPS - Hypertext Transfer Protocol Secure.
URL - Uniform Resource Locator- czyli adres, który wpisujemy w okienko przeglądarki. Każdy adres ma swoje składowe. Przykład: https://cotenfrontend:443/index.html?

  • https:// - protokół jakim się posługujemy
  • cotenfrontend.pl - to nazwa hosta (adres serwera)
  • :443 - port (nie widzimy tego w przeglądarce gołym okiem;))
  • / - super ważny - root
  • index.html - ścieżka dostępu do zasobu
  • ?parametr=wartość

DNS - Domain Name System - jest to rozproszony system nazw, dzięki króremu wpisujemy słownie adres strony a on przetłumacza go na jej adres IP
Przeglądarka parsuje i renderuje stronę = sprawdza składnie strony i zamienia kod HTML na to co widzimy, załadowując wszystkie css i inne zasoby.
Serwer jest w trybie request-response, czyli przeglądarka wysyła zapytania i dostaje odpowiedź od serwera, ale czasami otrzymuje błędy.

  • 1xx - hold on - poczekaj jeszcze chwilę, za moment dostaniesz odpowiedź
  • 2xx - here you are - wszystko ok
  • 3xx - go away - pojawia się w momencie, gdy szukamy czegoś czego nie ma juz, ale przekierowujemy tam gdzie powinieneś się dostać
  • 4xx - you fucked up - błąd po stronie przeglądarki- np. 404
  • 5xx - I fucked up - błędy po stronie serwera i związane z jego obsługą

Google Chrome umożliwia nam podgląd jakie mamy kody podczas wizyty na danej stronie. Robimy to za pomocą DevToolsów w zakładce Network.

GIT

GIT - jest to system kontroli wersji, który został stworzony przez Linusa Torvaldsa- twórcy Linuksa.

Po co jest potrzebny GIT?
  • projekt grupowy - widzimy nawzajem zmiany
  • kontrola nad wersją projektu
  • mozliwość rozbudowy różnych wersji projektu
  • przegląd zmian i wracanie do nich
Konfiguracja GIT-a
      git --version - sprawdzanie wersji gita
      git config --global user.name/"FirstName SurName"/
      git config --global user.email/email/
  • get name/get email - pobieranie danych o uzytkowniku i emailu
  • git init - tworzenie repo
  • ls -a - pokazywanie wszystkich plików
  • commit - stan projektu na daną chwilę i opis tego co zmieniło się od ostatniej wersji
  • gitk - pokazuje calą historię komunikatów
  • clear - czyści terminal
Commitowanie zmian
      git status
      git add - dodanie katalogu w ktorym sie znajdujemy do repo
      git add . - dodanie plików z danego katalogu
      initial commit - inicjowanie commita
      git commit-m ""- opis zmian

Git lens - narzędzie do podglądania repo i zmian
git w VS - commitowanie z poziomu VS
