2. HTML

HTML to skrót od HyperText Markup Lanaguage. Jest to język znaczników, a nie programowania. Składa się ze znacznikó, tagów, elementów.

2.1. Budowa znacznika:

<znacznik atrybut="wartość">treść</znacznik>

semantyczny= mający znaczenie

2.2. Znaczniki
  • <main> - główna część dokumnetu
  • <section> - sekcje dokumentu, pod względem logicznego podziału
  • <article> - samodzielne treści, artykuły w gazecie
  • <header> - nagłówek dokumentu, sekcji, artykułu
  • <footer> - stopka dokumnetu, sekcji, artykułu
  • <h1-h6> - tesktowe nagłówki tytułowe- dokumentu, sekcji, artykułu
  • <p> - paragraf, akapit

3. CSS

CSS to Cascading Style Sheets- kaskadowe arkusze stylów. Odpowiadają za prezentacje, wygląd elementow HTML na ekranie, wydrukach, czytnikach. Ma swoje reguły.

3.1. Budowa reguły CSS:

selector {
właściowść: wartość;
}

Sposoby stylowania:
  • w pliku html- w ramach znacznika <head> wpisujemy znacznik <style>
  • w pliku css- dzięki zewnętrznemu skoroszycie stylów <link href="style.css" rel="stylesheet" type="text/css">, co umieszczamy w <head> pliku index.html

1. Kanban- Japońskie planowanie

Ta metodologia należy do zwinnego podejscia organizacji pracy, jest po to aby śledzić swoje działania w projekcie jak i może być formą motywacji, gdy codziennie będziemy mogli zadania przerzucać. Narzędziem do tego może być np. Trello.

  • TO DO- lista to zrobienia
  • IN PROGRESS- lista zadań aktualnie realizowanych, zalecane jest aby zajmować się jednym zadaniem w danym czasie
  • DONE- lista zadań które zostały zrealizowane

często w projektach IT jest również TESTING- czyli zadania oddane do testów. Dobrą praktyką jest tworzenie również DoD- czyli Definition of Done, na podstawie zadań.

2. HTML II- atrybuty i znaczniki

  • <br/>- przenosi treść do nowej lini, jest to znacznik samozamykający
  • <b>- pogrubienie tekstu- wyróżnik wizualny
  • <strong>- pogrubienie tekstu, podkreśla wysoką rangę danego tekstu- wyróżnik semantyczny
  • <i>- pochylenie tekstu- wyróżnik wizualny

5. Jak działa przelądarka

HTTP - Hypertext Transfer Protocol - protokół, którego używa przeglądaka
Jest również HTTPS - Hypertext Transfer Protocol Secure.
URL - Uniform Resource Locator- czyli adres, który wpisujemy w okienko przeglądarki. Każdy adres ma swoje składowe. Przykład: https://cotenfrontend:443/index.html?

  • https:// - protokół jakim się posługujemy
  • cotenfrontend.pl - to nazwa hosta (adres serwera)
  • :443 - port (nie widzimy tego w przeglądarce gołym okiem;))
  • / - super ważny - root
  • index.html - ścieżka dostępu do zasobu
  • ?parametr=wartość

DNS - Domain Name System - jest to rozproszony system nazw, dzięki króremu wpisujemy słownie adres strony a on przetłumacza go na jej adres IP
Przeglądarka parsuje i renderuje stronę = sprawdza składnie strony i zamienia kod HTML na to co widzimy, załadowując wszystkie css i inne zasoby.
Serwer jest w trybie request-response, czyli przeglądarka wysyła zapytania i dostaje odpowiedź od serwera, ale czasami otrzymuje błędy.

  • 1xx - hold on - poczekaj jeszcze chwilę, za moment dostaniesz odpowiedź
  • 2xx - here you are - wszystko ok
  • 3xx - go away - pojawia się w momencie, gdy szukamy czegoś czego nie ma juz, ale przekierowujemy tam gdzie powinieneś się dostać
  • 4xx - you fucked up - błąd po stronie przeglądarki- np. 404
  • 5xx - I fucked up - błędy po stronie serwera i związane z jego obsługą

Google Chrome umożliwia nam podgląd jakie mamy kody podczas wizyty na danej stronie. Robimy to za pomocą DevToolsów w zakładce Network.

2. Display, wymiary i overflow

Internet jest w 2D;), w tej lekcji jest mowa o znacznikach <div>, które często są nadużywane. W ramach ich mozemy stosować poniższe rzeczy.

  1. display:block; - jest to takie ułożenie że dwa elementy sa ułozone w kolumnie
  2. display:inline; - jest to takie ułożenie że dwa elementy sa obok siebie i wypełniaja stronę. Jeśli jest jeden element to wtedy box sciupia się do takiej wielkości jaka jest treść
  3. display:inline-block; - jest to taka właściwość, w ramach której mozemy zastosować wymiary. Elementy ułoża się wtedy w boxie o ustalonych rozmiarach
  • width/hight - które spowodują ze elementy są wrażliwe na te wielkości i dopasowują sie do nich.
  • min/max width i min/max hight
  1. overflow
  • hidden - przepełniona wartość pozostanie ukryta
  • visible - wartość domyślna - treść nie jest obcinana, gdy wychodzi poza pole
  • scroll - również nam ukrywa ale możemy podejrzeć text za pomoca suwaka
  • auto - jeśli treść wychodzi poza pole to również pojawia nam sie opcja suwaka
  • initial - używa widocznej wartości domyślnej
  • inherit - (dziedziczenie) ustawia przepełnienie na wartość jego elementu nadrzędnego

3. Jednostki

  • px - podstawowa jednostka - piksel, np 1px
  • % - procenty, np. 50% danego elementu
  • vw - viewport width, czyli jest to procent z całego okna
  • vh - viewport height, czyli jest to procent z całego okna

4. Box model (padding, border, margin)

jest to niewidzialny prostokąt którym są otoczone wszystkie elementy.

4.1. padding - wewnętrzny margines

- jeśli chcemy zapisać je w jednej lini to musimy wartości wpisać według wskazówek zegara
-jeśli podamy tylko dwie wartości- traktujemy je wtedy jako wartości lustrzane, czyli dla top i bottom będzie taka sama wartość i dla left i right będzie taka sama

4.2. border - zewnętrzny margines- obramowanie

- width - określamy grubość
- style - jaki rodzaj chcemy używać -solid/dotted/dash/double/groove/ridge/inset/outset
- color - kolor
- radius - zaokrąglenie

margin - odległośc od innych elementów, zapisujemy jak w paddingu

- auto - wartość która jest różna w zależności jaki jest display.
- block- elementy ułożą się na środku body, w równych odstępach, ale musi mieć ustaloną szerokość, zeby wiadomo było do czego ma centrować.
- inline- nic nam nie da przyjęcie wartości auto

Aby móc brac pod uwagę całą szerokośc naszego elementu (wraz z paddingiem, borderem i marginem, ale bez zmieniania właściwoścci dla wszystkiego

8. JS:Typy danych i immutability

Zmienna to nie wartość, ona jedynie wskazuje nam na jakieś wartości, które przyjmujemy. W każdym języku programowania sa zmienne, w JS nazywamy je camelCasem. Zmienne są jak młody pelikan- łykną wszystko.

8.1.Zmienne w JS
  • let - zmienne, których połączenie możemy przypinać, ale ich nie definiujemy, wprowadzamy je tylko raz

let isHappy = false;
isHappy = tak;
isHappy = true;

  • const - zmienne mające stale połączenie z jedną wartością

const name = 'Mariola';
const age = 28;
przykład gniazdka i ładowarki

  • var
8.2. Typy proste w JS = primitive values
  • string - 'tekstowy'
  • number - liczbowy
  • bigint - liczba powyżej 2 do 53
  • boolen - wartości logiczne true/false
  • null - wartośc celowo niezdefiniowana
  • undefined - brak definicji
  • symbol - tworzy unikalne opakowanie

Sprawdzanie jakiego typu jest zmienna- typeof nazwa zmiennej. Ważne jest to że nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenie między nimi a zmiennymi.=Immutability=niezmienność

9. BEM = Block Element Modifier

Powstał przy pracy nad wysszukiwarką w Rosji, polega na spejalnym ukladzie nazw klas.

9.1 Dlaczego powstał?
  • problem z logicznym nazywaniem klas
  • brak ujednoliconego sposobu stylowania
  • importanty, nadpisywanie rzeczy
  • brak narzędzia odwzorowania struktur html w css
  • stylowanie kilkoma klasami, powtarzające się nazwy
9.2. Z czego się składa BEM
  1. Block - nadrzędny element
  2. Element - część bloku (nie musi być bezpośrednim dzieckiem)
  3. Modifier - modyfikator - jeśli element występuje w kilku wariacjach tworzymy dodatkową klasę
9.3. Sposób zapisu

block__element--modifier, aby lepiej to wyjaśnić poniżej przykład ze zdjęciem

9.4. Co warto zapamiętać?
  1. szukamy elementów na stronei i dzielimy ją na bloki
  2. myślimy o elementach składowych- plan strony i jej analiza-selekcja
  3. tworzymy dwie klasy jeśli element ma modyfikator (w modyfikatorze zmieniamy tylko unikalne dla niego wartości, nie przepisujemy rzeczy z klasy bazowej!!!)
